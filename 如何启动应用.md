# 如何启动应用 - 日常使用指南

## ⚠️ 重要说明

**开发模式（当前方式）的特点：**
- 应用只在终端运行时才运行
- **关闭终端窗口后，应用会停止**
- 每次使用都需要重新启动
- 这种方式适合开发和测试

**生产环境部署（后续可选）：**
- 应用会一直运行在服务器上
- 不需要每次都启动
- 适合正式使用场景

---

## 🚀 方法一：使用启动脚本（推荐 - 最简单）

### Mac/Linux 用户：

1. **打开终端**
2. **运行启动脚本：**

```bash
cd "/Users/Zhuanz/cursor/用线计算器"
./启动应用.sh
```

或者直接双击 `启动应用.sh` 文件（需要赋予执行权限）

### Windows 用户：

双击 `启动应用.bat` 文件即可

---

## 🚀 方法二：手动启动（标准方式）

每次需要使用应用时，按以下步骤操作：

### **步骤1：打开终端**

- **Mac**: 按 `Command + 空格`，输入 `Terminal`，回车
- **Windows**: 按 `Win + R`，输入 `cmd`，回车

### **步骤2：进入项目目录**

```bash
cd "/Users/Zhuanz/cursor/用线计算器"
```

### **步骤3：修复环境变量（Mac用户需要，Windows用户可跳过）**

```bash
export PATH="/usr/local/bin:$PATH"
```

### **步骤4：启动应用**

```bash
npm run dev
```

### **步骤5：等待启动完成**

您会看到类似以下的输出：

```
服务器运行在 http://localhost:3000
  VITE v5.0.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### **步骤6：打开浏览器**

访问：**http://localhost:5173**

---

## ⏸️ 停止应用

### 在终端中按：
```
Ctrl + C
```

或者直接关闭终端窗口

---

## 📝 快速启动命令（一键复制）

如果您想快速启动，可以直接复制以下命令到终端：

### Mac/Linux:
```bash
export PATH="/usr/local/bin:$PATH" && cd "/Users/Zhuanz/cursor/用线计算器" && npm run dev
```

### Windows:
```bash
cd "C:\Users\Zhuanz\cursor\用线计算器" && npm run dev
```

（注意：Windows用户需要根据实际路径调整）

---

## 🔧 如果启动脚本无法运行（Mac/Linux）

如果双击脚本文件无法运行，请在终端中执行：

```bash
cd "/Users/Zhuanz/cursor/用线计算器"
chmod +x 启动应用.sh
./启动应用.sh
```

## 💡 让启动更方便的小技巧

### 方法1：创建终端别名（Mac/Linux）

在 `~/.zshrc` 或 `~/.bashrc` 文件中添加：

```bash
alias 启动计算器='export PATH="/usr/local/bin:$PATH" && cd "/Users/Zhuanz/cursor/用线计算器" && npm run dev'
```

然后运行：
```bash
source ~/.zshrc  # 或 source ~/.bashrc
```

以后只需要输入 `启动计算器` 即可启动应用。

### 方法2：创建桌面快捷方式（Windows）

1. 右键点击 `启动应用.bat`
2. 选择"发送到" → "桌面快捷方式"
3. 以后双击桌面快捷方式即可启动

### 方法3：添加到系统启动项（Mac）

可以创建 Launch Agent，让应用在系统启动时自动运行（但这不是推荐方式，因为开发服务器不应该一直运行）

## ❓ 常见问题

### Q1: 为什么关闭终端后应用就停了？

**A:** 因为这是开发模式。开发服务器需要在终端中运行。如果想应用一直运行，需要部署到生产环境。

### Q2: 可以后台运行吗？

**A:** 可以，但不太推荐。开发模式应该在前台运行以便查看日志。如果确实需要后台运行：

```bash
# Mac/Linux
nohup npm run dev > app.log 2>&1 &

# Windows (需要使用其他工具)
```

### Q3: 启动很慢怎么办？

**A:** 
- 第一次启动会比较慢（需要编译）
- 确保没有其他程序占用 3000 或 5173 端口
- 如果还是很慢，可以分别启动前后端查看具体哪个慢

### Q4: 每次都要输入这么多命令吗？

**A:** 使用启动脚本（方法一）最简单，只需要双击或运行一个命令。

## 📌 日常使用流程总结

**最简单的方式：**

1. 打开终端
2. 运行 `./启动应用.sh` （Mac）或双击 `启动应用.bat`（Windows）
3. 等待启动完成
4. 打开浏览器访问 http://localhost:5173
5. 使用完毕，在终端按 `Ctrl+C` 停止

**就这么简单！**
